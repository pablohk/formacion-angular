Datos Padre‑Hijo

Aquí se trabaja el flujo de información entre componentes padre, hijo y nieto.

El contenedor principal renderiza los componentes padre y muestra dos enfoques: prop drilling y servicio compartido.

Prop drilling

El contenedor pasa la información hacia el componente Padre y, desde el nieto, se emiten eventos hacia arriba.

Servicio compartido

Uso de un servicio para compartir estado entre componentes sin prop drilling.
